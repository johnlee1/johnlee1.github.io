prayforhana.org
about
Hana is a web application that provides a platform for sharing prayer requests. It is an ongoing project I am working on with some friends. The goal is to make an application that connects people through prayer but without replacing real-life community. I will be updating the app every other week with new features, ui updates, or bug fixes.
target users
Hana is designed for those who want to pray. An expected common use case of this application is learning how to pray for others when it is not possible to directly ask them for prayer requests. This could be helpful when praying for missionaries, missions organizations, world events, or people groups.
tone
The intended tone we are trying to achieve with Hana can be described with the following: biblical, informative, encourages reflection, simple. We want to design a ux/ui with this in mind.
tech infrastructure
The frontend is an angular 2 app served by nginx. The api backend is a node app using hapi. Both nginx and node live in docker containers that are deployed with aws ecs. The nodejs app is connected to a mongodb database provided by mlab. DNS is handled with namecheap and aws route53. Version control is handled with git and github.
looking forward
I want to create a more automated process for developing, building, and deploying the applications by looking into options for ci/cd. In regards to infrastructure, currently the project is set up in a way that does not allow the frontend and backend to scale independently of each other since they are both configured to live on the same server instance. In the future it could possibly be beneficial to have a more flexible microservices setup that allows independent horizontal scaling.
